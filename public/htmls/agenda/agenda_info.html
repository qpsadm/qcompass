LaravelによるWebアプリ作成①　Laravelを準備する

1　Laravelを準備する

1.1　はじめに

  1. 必要な事前知識とスキル：HTML・CSS・JavaScriptによる静的Webページの制作スキル、PHPプログラミングスキル、MariaDBおよびSQLを用いたデータベース操作スキル。
  2. 教科書の動作環境：Laravel 12をベースとし、PHP 8.4に対応。
  3. サンプルコード・プロジェクトのダウンロード先：
    https://www.shuwasystem.co.jp/support/7980html/7527.html
  4. 「解凍」について、以下のWebページをご参照ください。
    https://www.shuwasystem.co.jp/support/kaitou.html
  5. サンプルコードの保存場所：
    ユーザープロファイル内に「ws_laravel」フォルダを作成し、Laravelの作業フォルダとして使用しましょう。

1.2　PHP開発の問題点

PHPなどのライトウェイトな言語※1には、本格的な言語とは異なる事情があります。それは、ライトウェイトな言語は誰でも手軽に扱えるため、プログラミング経験の浅い人が自己流でコードを書くケースが多いという点です。こうした自己流の開発は、将来的にさまざまな問題を引き起こす可能性があります。代表的な問題として、以下の3点が挙げられます。

  • セキュリティの問題
    データベースなどの情報を元に動的にWebを生成するようなサイトになると、外部からの攻撃への対策を考えなければいけません。セキュリティ対応するには、セキュリティ対策についての知識とプログラミング経験は欠かせません。
  • メンテナンスの問題
    自己流に書いたプログラムは、コーディングルールやデザインパターンなどあまり考えずに書いている場合が大半です。このようなプログラムは第三者への引継ぎができません。
  • 拡張性の問題
    自分1人で考えて作ったものというのは、最初から拡張性や汎用性などを考えて設計することはあまりありません。より快適に使ってもらえるようにシステムを改良や拡張する必要がでた場合、設計の必要性を感じます。
※1　ライトウェイトな言語とは、軽量プログラミング言語（Lightweight Language（略してLL）とも呼ばれます。短い記述で処理を実現でき、手軽に取り扱うことができるプログラミング言語です。 PythonやRubyはLLの代表格です。 LLはCやJavaなどを使うまでもないような、ちょっとした処理を書く際に利用するととても便利です。

1.3　フレームワークの導入

上記のような問題の多くは、適切なシステム設計を行うことで回避可能です。 つまり、セキュリティ・メンテナンス性・拡張性などを最初から考慮して設計されたシステムに基づいてプログラムを作成すれば、誰でも堅牢で分かりやすい コードを書くことができます。 このような開発において多用される、さまざまな機能や仕組みを提供するものが「フレームワーク」と呼ばれます。


1.4　フレームワークとライブラリとの違い

フレームワークとは

フレームワークとは、「枠組み」「骨組み」「構造」といった意味を持つ言葉で、自由度の高いプログラミングの世界に一定のルールや制約を与えてくれるものです。 プログラミングにおいては、特定のオペレーティングシステム向けのアプリケーションにおける標準的な構造を実装するための、クラスやライブラリの集合体を指します。 つまり、汎用的に必要とされる機能をまとめた、開発のための枠組みや骨組みのことです。
フレームワークには、実装負荷の軽減、保守性の向上、セキュリティの確保、規則に則ったコードの記述、そして高い可読性といった特長があります。 ただし、素のPHPと比べると自由度は低く、関数・クラス・メソッドの名称などはフレームワークの規則に従わなければ正しく動作しないというデメリットもあります。

ライブラリ

ライブラリとは、汎用性の高い複数のプログラムをひとまとめにしたもので、クラスや関数などの部品がまとまって提供されるソースコードの集合体を指します。 システム内の要所要所で活用できる、便利な機能群というニュアンスを持っています。

「フレームワークはシステム全体の枠組み」、「ライブラリは特定機能を持つ小さな部品」と覚えておくとよいでしょう。

1.5　Webフレームワークの役割

Webフレームワークの最も基本的な役割は、HTTP通信の制御です。 HTTPを通じてWebサーバーに必要なデータをリクエストし、そのレスポンスを取得・返却する処理がコアとなっています。 さらに、多くのWebフレームワークは、以下の3つの主要なパーツで構成されています。

  • モデル（Model）：データベースとのやり取りを担当する
  • ビュー（View）：Webページを動的に生成し、表示する
  • コントローラー（Controller）：HTTPリクエストを受け取り、モデルやビューに指示を出す

この3つの要素を持つWebフレームワークは「MVCフレームワーク」と呼ばれます。 つまり、MVCフレームワークでは、プログラムファイルが主に「表示（View）」「データ（Model）」「制御（Controller）」の3つに分けられていると理解してください。

1.6 主なPHPのWebフレームワーク

PHP開発の際によく使用されるPHPフレームワークは複数あります。主な人気度が高いPHPのWebフレームワークには、以下のようなものがあります。

  • CakePHP：
    2005年に登場しオープンソースで開発されたPHPフレームワークです。Ruby on Railsの影響を強く受けており，Webアプリケーションを高速に開発出来ます。現在でもフリーランス求人・案件数が一番多く、人気のPHPフレームワークです。
  • Symfony：
    2005年に登場したMITライセンスに基づくオープンソースのPHP5用フレームワークです。保守性と安定性を重視した構造になっており、CakePHPと同様Ruby on Railsの影響を受けています。2021年現在、LaravelやCakePHPと比較すると、開発エンジニア人口が少ないです。
  • Laravel：
    Laravelは、2012年に登場したMVCフレームワークで、近年PHPエンジニアの間で急速に人気を高めています。

現在、PHPフレームワークの中で最も注目されているのがLaravelです。 その背景には、コードの記述が非常にしやすいこと、日本語のドキュメントが豊富であること、ヘルパー関数をはじめとするライブラリが充実していることなどが挙げられます。 中でも、学習コストの低さはLaravelの大きな特徴のひとつです。

参考サイト：PHPフレームワークのまとめ【2022年版】
https://freelance-start.com/articles/70#title1

1.7 LaravelとPHPバージョンについて

Laravelは、2025年2月にver. 12がリリースされました。Laravel 12は、PHPのver.　8.2から8.4で動作するように作られています。

2　Laravelのインストール

Laravelフレームワークを使用する場合、Webサーバー、PHP、Composer、データベースの環境を準備する必要があります。初心者にとっては、XAMPPを使って簡単にLaravelの開発環境を構築することができます。Macパソコンの場合は、MAMPをお勧めします。Laravelの開発環境を構築する手順は、教科書P6～P13までご参照ください。

2.1　Composerについて

Composerとは、PHPのパッケージ依存管理ツールで、PHPの開発で使用するライブラリ（パッケージ）をまとめて管理してくれます。Composerは、ライブラリ間の依存関係を解決してくれるため、開発者の負担を大きく軽減してくれます。Composerが対応しているPHPのバージョンは「PHP 5.3.2」以上です。Composerのインストール手順については、教科書P7～P9をご参照ください。

2.4　Laravelをインストールする

Composerが準備できたら、次のコマンドを使ってLaravelをインストールできます。

コマンド：composer global require Laravel/installer

※　上記の命令では、Laravelの本体をインストールするのではなく、Laravelのインストーラをインストールしたことです。

「Laravelを利用する」とは、単にLaravelをインストールして使うことではなく、専用のLaravelインストーラを使って、必要に応じてLaravelプロジェクトを新規作成することを指します。

2.5　Laravel開発の手順

Laravelの開発手順は、基本的にコマンドラインから実行して行います。開発の流れは、教科書のP9～P10をご参照ください。

2.6　Laravelプロジェクトの作成

Laravelのプロジェクト作成手順については、教科書のP10～P13をご参照ください。

2.7　Herdを活用する

コマンドを使ってLaravelプロジェクトを作成できますが、コマンド操作が苦手な方にとってはハードルが高く感じられるかもしれません。 現在のLaravelでは、開発専用の支援ツール「Herd」を導入することで、PHP、開発用サーバー（nginx）、サーバーサイド開発に必要なNode.jsなどを一式まとめて簡単に準備できます。 Herdを使えば、Laravelのインストールやプロジェクトの作成・管理をGUIベースで行うことが可能です。「Herd」のインストール方法およびLaravelプロジェクトの作成・管理手順については、教科書のP13～P20をご参照ください。

「Herd」は非常に便利なツールですが、高度な機能を利用するにはProバージョンの購入が必要です。 本講座では、無料版（フリーバージョン）を使用して、Laravelの開発環境の構築およびプロジェクト作成の手順を詳しく解説します。詳細的な手順は、P13～P20までご参照ください。

3　XAMPPにデプロイする

3.1　作成したプロジェクトをXAMPPに公開

Laravelアプリケーションは、XAMPPを使用して公開することも可能です。 その基本的な手順は、以下の通りです。

  1. Laravelプロジェクトのフォルダ「laravelapp」をコピーします。
  2. コピーしたフォルダを「htdocs」ディレクトリに貼り付けます。

アプリケーションへのアクセスアドレスは、以下のようになります。

「http://localhost/laravelapp/public」

※URLを短くする方法もありますが、ここでは割愛します。

4.3　Laravelの命名規則について

スムーズに開発を進めるためにも命名規則に沿って名前をつけることは大事です。Laravelをはじめて学習する方は最初に命名規則を抑えておくと、プロジェクト全体に統一感が出ていいと思います。

プログラミングにおいて複数キーワードの名前の付け方としてアッパーキャメル記法、ローワーキャメル記法とスネーク記法という記述方法があります。

１　アッパーキャメル記法

先頭を大文字で書き始め、複合語のスペースをなくし要素の先頭を大文字にします。

[例] user + parameter ⇒ UserParameter

Laravelでの使用例：
UserMsg.php（モデル）、UserMsgController.php（コントローラ）

２　ローワーキャメル記法

先頭を小文字で書き始め、複合語のスペースをなくし要素の先頭は大文字にします。

[例] user + parameter ⇒ userParameter

Laravelでの使用例：userProfile()（メソッド）

３　スネーク記法

単語と単語の区切りにアンダースコアを使う記述方法をスネーク記法と呼びます。

[例] user + parameter ⇒ user_parameter

Laravelでの使用例：access_rankings（テーブル名）、$user_name（変数名）、user_admin.blade.php（Bladeのファイル名）、access_rankings（テーブル名）
